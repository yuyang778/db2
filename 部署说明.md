# HTML页面保护功能部署说明

## 一、自动部署方法（推荐）

我们提供了两种简单的部署方法，您可以任选其一：

### 方法一：使用一键部署工具

1. 双击运行`一键部署保护.bat`文件
2. 等待部署完成，脚本会自动为所有HTML文件添加保护功能
3. 完成后会显示详细的部署报告

### 方法二：使用PowerShell脚本

1. 右键点击`保护HTML页面.ps1`文件
2. 选择"使用PowerShell运行"
3. 等待部署完成

## 二、部署功能说明

自动部署工具会完成以下任务：

1. 扫描当前目录及所有子目录中的HTML文件
2. 为每个HTML文件添加保护脚本引用
3. 为所有链接添加`target="_blank"`属性（使其在新窗口打开）
4. 将保护脚本`page-protector.js`复制到所有包含HTML文件的目录中

## 三、手动部署方法

如果自动部署工具无法正常工作，您可以手动完成以下步骤：

1. 将`page-protector.js`文件复制到每个包含HTML文件的目录中
2. 在每个HTML文件的`</head>`标签前添加以下代码：
   ```html
   <script src="page-protector.js"></script>
   ```
3. 为每个HTML文件中的链接添加`target="_blank"`属性

## 四、验证部署

部署完成后，请打开网站并验证以下功能：

1. 无法右键点击页面
2. 无法复制页面内容
3. 点击链接时在新窗口打开
4. 尝试使用开发者工具时受到限制

## 五、注意事项

- 部署工具不会对已经添加了保护功能的文件重复处理
- "返回首页"等导航链接不会设置为在新窗口打开
- 如果PowerShell脚本无法运行，请尝试以管理员身份打开PowerShell并执行：`Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
